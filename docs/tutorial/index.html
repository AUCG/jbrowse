The Config File

A JBrowse config file exists to tell the JBrowse tools about a database and the tracks within the database. If all of your data is in flat files, and your feature flat files (BED, GFF) are separated so that there's only one file per track, you don't need a config file. However, a config file is a convenient way to specify the details of how JBrowse should treat a track, so you may want to use a config file anyway, especially if you expect to import new versions of your data on an ongoing basis.

The syntax of the config file is JSON. It was originally the GBrowse config file syntax, but JBrowse really needed multiple levels of nesting and an array syntax, so it's JSON. However, it does contain some of the same settings as the GBrowse config file, especially at the beginning. A JBrowse config file starts out just like a GBrowse config file, with a slightly different syntax:

{
  "description": "Volvox Example Database",
  "db_adaptor": "Bio::DB::SeqFeature::Store",
  "db_args": { "-adaptor": "memory",
               "-dir": "docs/tutorial/data_files" },
